## 막힌 지점 / 보강한 것
- 🔴 **Promise.all 회귀(핵심):** "잘 모름"으로 막힘. 순차 `const a=await fetchA(); const b=await fetchB()`에서 **"b도 이미 시작됨"으로 오답** — 이게 핵심 오해. `await`가 *다음 줄 진입 자체*를 막아 b는 a 끝난 t=1에야 호출(→2초)임을 단계 유도로 회복. **재각인: 병렬성은 `Promise.all`도 await 개수도 아니라 "await 전에 다 호출해 in-flight로 만든 출발 시점"에서 나온다.** 회복 후 사용자가 스스로 ①분리호출 2개=Promise.all 타이밍 동일 / 차이는 에러(fail-fast·미await reject의 unhandled rejection) ②동시 출발=호스트가 네트워크 대기 떠안아 겹침 — 두 심화 질문 발의·자가 도출. → m4→3, **1일 리셋(6-25)**.
- ⭐ **정밀 질문(사용자 자력 발의):** "콜 스택을 가진 주체가 누구냐 — 언어? 엔진? 호스트?" → **3층 정리:** 언어(JS/ECMAScript=사양, 스레드 개념 없음) / 엔진(V8·SpiderMonkey·JSC=코드 실행 스레드 1개=콜스택 1개) / 호스트(브라우저·Node=엔진 임베드 + API + 이벤트 루프 + I/O 스레드). **"이벤트 루프·`setTimeout`·`fetch`는 엔진이 아니라 호스트가 제공"**(V8만 떼면 콜스택·힙뿐). 같은 V8이 Chrome·Node 양쪽 — 호스트만 다름. process 페이지에 박제. 페이지 claim을 스스로 의심해 정밀화 — mastery 4의 결.
