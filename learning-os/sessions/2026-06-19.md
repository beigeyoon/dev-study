## 진행한 것
- **Promise 신규 학습 (ingest + 파인만 통과, mastery 3).** 백지 암송 없이 앵커 추론으로 한 조각씩 재구성:
  - 상태 3종 pending→fulfilled/rejected, settled 불변.
  - **resolve=동사(행동), fulfilled=명사(상태)** 구분. resolve에 Promise 넘기면 "resolved≠fulfilled" 떡밥.
  - resolve/reject = 엔진이 인자로 건네주는 함수(executor 패턴). **값은 return이 아니라 인자로 전달.**
  - 버튼 누르는 주체 = 비동기 작업을 실제 수행하는 백그라운드 일꾼(Web API). fetch 내부의 숨은 executor.
  - `.then`은 등록만 → settled 순간 콜백이 마이크로큐로. `.then`은 새 Promise 반환(체이닝) + 자동 flatten.
  - **강점:** 상태·값을 박제 → 콜백이 타이밍 경쟁에서 자유로워짐(늦게 .then 붙여도 안 놓침).
- 기존 5개(async/await·microtask·event-loop·macrotask·call-stack·promise-all) 전부 양방향 연결.

## 막힌 지점 / 보강한 것
- 통합 파인만 처음엔 "잘 모르겠다" → 빈칸 채우기로 쪼개니 1~3단계 스스로 재구성.
- 오해 3개 교정: ① resolve/reject가 상태인 줄(→행동), ② resolve가 값을 return하는 줄(→인자), ③ 이미 fulfilled된 Promise에 .then 늦게 붙이면 콜백 안 가는 줄(→즉시 마이크로큐로, 오히려 최속).
