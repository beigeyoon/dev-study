## 진행한 것
- **HTTP 복습 (첫 간격) — ✅ feynman 통과, learning→understood, m2→3, review_due→6-26.**
  - 앵커: `await fetch(".../users/42")`가 선 위로 보내는 텍스트를 직접 그려라.
  - **요청 3층(시작줄/헤더/바디)을 무힌트로 자력 재구성** (어제는 헤더를 코치가 유도했던 부분).
  - GET 바디: 처음 "바디"라 적었다가 **스스로 정정** — "GET은 딸려 보낼 데이터 없음→비움" (동사가 바디 유무를 가름).
  - **`Host` 헤더를 1차 원리로 도출:** 시작줄엔 path만, URL의 `api.example.com`이 빠짐 → "같은 건물(IP)에 여러 회사(도메인)→회사 이름 없으면 배달 불가" → `Host:` 헤더 = 버추얼 호스팅. (이름은 코치가 줌, 논리는 자력.)
  - 응답 첫 줄 = status code + 사유구. **fetch 404 함정 양방향 설명:** 404=fulfilled(답장 도착=네트워크 성공) / reject는 답장이 아예 못 올 때만(서버다운·DNS·오프라인) → `if(!res.ok)`로 직접 판별.
- **🆕 쿠키/세션/토큰 신규 학습 (network 2번째 점, learning·m2·feynman 미통과).**
  - **연결 먼저(§8):** HTTP stateless 트레이드오프 떡밥의 직접 후속 + 운반 수단이 HTTP 헤더(`Set-Cookie`/`Cookie`)임을 다리로 제시.
  - **유도 추론:** "두 번째 요청이 나를 어떻게 증명?" → **헤더에 신분증=쿠키** 자력 도출(오늘 배운 3층 즉시 활용) → 출처=로그인 응답(`Set-Cookie`)·운반=브라우저 자동(`Cookie`) 추론.
  - **세션 vs 토큰 자력 압축:** "세션=서버가 상태 하나하나 기억(stateful) ↔ 토큰=정보가 토큰 자체에(stateless 가능)." "누가 상태를 기억하나" 축을 스스로 잡음.
  - **트레이드오프:** 확장성(stateless)↔즉시 무효화(stateful)는 동시에 깔끔히 못 가짐 → 하이브리드(짧은 access token + 서버 추적 refresh token). 실사용 예시(세션=은행·전통 웹앱 / 토큰=모바일·SPA·마이크로서비스·OAuth / 둘 다=큰 서비스)까지.
- **🆕 프로세스 vs 스레드 추가 학습 (os 영역 개방, learning·m2·feynman 미통과).** ← 사용자 요청 "기존에 안 다룬 카테고리".
  - **연결 먼저(§8):** 그동안 반복 등장한 "JS 싱글 스레드"·"동시성≠병렬성"·"Web Worker 복사 vs 참조"의 밑바닥이 프로세스/스레드임을 다리로 제시.
  - **유도 추론:** 프로세스가 OS한테 받는 것 → "저장 공간"(메모리) 자력 / 스레드 → 이미 쓰던 단어로 자력 / 스레드끼리 메모리 공유 추론.
  - **race condition 자력 트레이스:** `balance=100`, A(+50)·B(-30) → 읽기/계산/쓰기 3단계 인터리빙을 혼자 따라가 **최종 70·업데이트 유실·비결정적(실행마다 다름)**까지 도출. "실행 시간/속도에 따라 달라진다"를 스스로 말함.
  - **핵심 reframe:** "JS 싱글 스레드 = race condition·락 면제 = 선물"(병렬 포기 ↔ 동시성 버그 면제 트레이드오프).
  - **Web Worker 복사 자력 연결:** "원본 공유하면 race 터지니까 일부러 복사·격리" → 며칠 전 떡밥 해소. SharedArrayBuffer 시 락 필요(떡밥).

## 막힌 지점 / 보강한 것
- 쿠키/세션/토큰: **토큰 무효화는 스스로 못 풀었음**("모르겠어") — 이게 정확히 약점이 드러나는 지점("지울 게 없다=깔끔히 무효화 불가")임을 코치가 reveal. 학습 신호로 처리.
- **압축의 비대칭:** 세션의 대가(stateless 포기)는 적었으나 **토큰의 대가(즉시 무효화 불가)는 빠뜨림.** 코치가 대칭 보강. 다음 복습에서 양쪽 대가를 다 드는지 점검.
- 첫 접촉 = 유도된 학습 → learning·m2·feynman 미통과로 정직하게 매김(HTTP가 어제 그랬듯). 6-24 첫 간격에서 무힌트 점검.
- 프로세스/스레드: 락/뮤텍스(race 해결책)는 이름만, 트레이스는 자력. 첫 접촉이라 learning·m2. 다음 복습에서 프로세스(격리)vs스레드(공유) 대비 + "싱글스레드=선물"을 무힌트로 세우는지 점검.
